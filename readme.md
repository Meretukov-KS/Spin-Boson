# Сравнение TCL2 и HEOM для спин-бозонной модели

Проект реализует численное сравнение двух методов описания открытых квантовых систем:

- **TCL2 (Time‑Convolutionless, 2‑й порядок)** – приближённое интегро-дифференциальное уравнение для редуцированной матрицы плотности спина.
- **HEOM (Hierarchical Equations of Motion)** – точный метод, учитывающий не‑марковскую динамику через иерархию вспомогательных операторов плотности.

Расчёты выполнены для спин-бозонной модели с лоренцевской спектральной плотностью  
\( J(\omega) = \lambda_s\,\gamma\,\omega/(\omega^2 + \gamma^2) \).

## Физическая модель

**Система**: двухуровневая система (кубит) с гамильтонианом

$$
H_S = -\frac{\Delta}{2}\,\sigma_x + \frac{h}{2}\,\sigma_z
$$

**Окружение**: бозонное поле, описываемое спектральной плотностью

$$
J(\omega) = \lambda_s \,\frac{\gamma\,\omega}{\omega^2 + \gamma^2}
$$

Полная связь система+окружение имеет вид $ H_{int} = \sigma_z \otimes \sum_k g_k (a_k + a_k^\dagger) $.  
В TCL2 константа связи масштабируется дополнительным фактором $\Lambda$:

$$
\lambda_{\text{TCL2}} = \Lambda^2 \lambda_s
$$

(позволяет исследовать область слабой связи, где TCL2 должен работать хорошо).

**Наблюдаемые**:

- средние значения $\langle\sigma_x\rangle,\langle\sigma_y\rangle,\langle\sigma_z\rangle$;
- чистота редуцированного состояния $\text{Tr}(\rho_S^2)$;
- длина вектора Блоха.

## Структура проекта

| Файл | Назначение |
| ------ | ------------- |
| `params.py` | Единый файл всех параметров модели (для обоих методов). |
| `numerical.py` | Реализация TCL2: вывод уравнений, численное решение методом TR‑BDF2, учёт памяти и обратной связи по населённостям мод. |
| `heom.py` | Решение HEOM с помощью библиотеки QuTiP (допускает выбор разложения Паде или Матсубары). |
| `compare.py` | Запускает оба метода, строит графики наложения (overlay) и разностей (difference). |

## Зависимости

Установите необходимые пакеты:

```bash
pip install numpy scipy matplotlib numba qutip
```

**Библиотеки**:

- numpy, scipy - основные вычисления, интегрирование,
- matplotlib - графики,
- numba - оптимизация за счет предкомпиляции часто используемых функций,
- qutip - реализация HEOM.

## Настройка параметров

Параметры как для HEOM, так и для TCL2 решения задаются через файл `params.py`. Параметры разбиты на 3 группы

- Общие для обоих методов:

| Параметры | Роль |
| ----- | ----- |
| `Delta` | Туннельная амплитуда (расщепление) |
| `H_FIELD` | Продольное магнитное поле |
| `lambda_s` | Сила связи в \(J(ω)\) |
| `GAMMA` | Ширина лоренцевского пика |
| `Temp` | Температура окружения (\[k_BT\] в единицах ℏ⋅частота) |
| `LAMBDA` | Дополнительный множитель для TCL2 \(λ_{TCL2}=Λ^2λ_s\​) |
| `T_MAX`, `T_POINTS` | Время интегрирования и число точек |
| `OMEGA_MAX`, `OMEGA_POINTS` | Сетка по частотам для TCL2 |
| `N_DEPTH`, `N_MATSUBARA` | Параметры сходимости HEOM |

## Запуск

Есть два режима работы: сравнение, счет.

1. Для первого надо запустить `compare.py`, который по очереди выполнит:

    1. Решение TCL2 из файла `numerical.py`,
    2. Решение HEOM из `heom.py`,
    3. Построит два рисунка:
        - `compare_overlay.png` – наложение кривых TCL2 и HEOM,
        - `compare_difference.png` – разности (TCL2−HEOMTCL2−HEOM) и сводка максимальных отклонений.

    Для запуска можно прописать в командной строке

    ```bash
    python compare.py
    ```

2. Для второго надо запустить `numerical.py`, который численно посчитает решение и построит графики динамики средних, частоты и т.д.

Для управления спектральной плотностью используется функция $J(\omega, t)$.

В результате выполнения создается новая папка в каталоге numerical, который будет находится по пути, указанному в `.env` файле. В катологе numerical будет создан новый каталог с названием времени выполнения кода и в нем будет 3 файла: график, `param.py` файл и текстовый файл, содержащий код функций $J$ и $N_I$.
